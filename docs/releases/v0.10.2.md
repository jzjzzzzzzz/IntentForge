# IntentForge v0.10.2 Release Notes

## Summary

v0.10.2 prepares IntentForge for PyPI distribution. No new CAD model families or breaking API changes are introduced.

## Changes

### PyPI Distribution

- Bumped package version to `0.10.2` in `pyproject.toml`.
- Added `[tool.setuptools.package-data]` so that benchmark prompt/expected JSON files and harness JSON/YAML data files are correctly bundled in the wheel and sdist. These files are required at runtime when installed from PyPI.
- All user-facing install instructions, error messages, and documentation now reference `pip install intentforge` and `pip install "intentforge[extra]"` rather than the editable source install form.

### GitHub Actions OIDC Trusted Publisher

- Added `.github/workflows/publish.yml`: triggers on `v*` tags, builds wheel and sdist, runs `twine check`, then publishes to PyPI via `pypa/gh-action-pypi-publish` using OIDC Trusted Publisher. No API keys are stored in repository secrets.

### Documentation

- Updated install instructions in README, docs/mcp.md, docs/product_demo.md, docs/llm_translator.md.
- Updated error message strings in CLI, API server, artifact server, and CadQuery generator to reference PyPI install form.

## Unchanged

- CAD generation logic (wall_mounted_bracket, l_bracket families) is unchanged.
- MCP tools, HTTP API server, LLM translator, technical harness, and product demo workflows are unchanged.
- All API contracts (ToolResponse envelopes, request_id, artifact refs, structured errors) are unchanged.
- No new CAD model family.
